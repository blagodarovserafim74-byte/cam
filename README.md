# IP Camera Viewer

Нормальная модульная структура проекта для дальнейшей разработки, а не один большой файл.

## Структура проекта

```text
cam/
├── main.py
├── requirements.txt
├── README.md
└── ip_camera_viewer/
    ├── __init__.py
    ├── app.py
    ├── config.py
    ├── models.py
    ├── services/
    │   ├── __init__.py
    │   └── camera_stream.py
    ├── ui/
    │   ├── __init__.py
    │   └── main_window.py
    └── utils/
        ├── __init__.py
        ├── image.py
        └── validators.py
```

## Что за что отвечает

- `main.py` — точка входа.
- `ip_camera_viewer/app.py` — запуск приложения.
- `ip_camera_viewer/config.py` — настройки интерфейса и потока.
- `ip_camera_viewer/models.py` — типы сообщений между потоком и GUI.
- `ip_camera_viewer/services/camera_stream.py` — подключение к камере и чтение кадров.
- `ip_camera_viewer/ui/main_window.py` — графический интерфейс.
- `ip_camera_viewer/utils/validators.py` — проверка URL камеры.
- `ip_camera_viewer/utils/image.py` — подготовка кадра для показа в окне.

## Почему такая структура лучше

- проще добавлять новые функции;
- легче искать ошибки;
- удобно тестировать части проекта отдельно;
- интерфейс не смешан с логикой камеры;
- можно дальше добавить запись видео, историю камер, настройки и логирование.

## Возможности

- поле для ввода ссылки на IP-камеру;
- кнопка **«Подключиться»**;
- кнопка **«Отключиться»**;
- область для показа видео;
- статус подключения;
- понятные ошибки при неверной ссылке или недоступной камере;
- корректная остановка без зависания окна.

## Поддерживаемые ссылки

Примеры:

```text
rtsp://login:password@192.168.1.10:554/stream
http://192.168.1.10:8080/video
```

Ссылку нужно вставлять в поле **«Ссылка на IP-камеру»** в верхней части окна.

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Windows

Проект рассчитан на запуск в Windows с обычным Python 3.10+.
Tkinter обычно уже входит в стандартную установку Python.
